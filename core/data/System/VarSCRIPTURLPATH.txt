#VarSCRIPTURLPATH
---+++ SCRIPTURLPATH{"script"} -- URL path of script
   * Expands to the base URL of scripts, without protocol or host
   * The optional =path= parameter can be used to add a =Web.Topic= (or =Web/Topic=, or just a web)
   * Any other parameters to the macro will be added as parameters to the URL
   * Example: =%<nop>SCRIPTURLPATH{"view" path="Cartoons.EvilMonkey"}%= expands to =%SCRIPTURLPATH{"view" path="Cartoons.EvilMonkey"}%=
   * Example: =%<nop>SCRIPTURLPATH{"view" path="Cartoons"}%= expands to =%SCRIPTURLPATH{"view" path="Cartoons"}%=
   * Example: =%<nop>SCRIPTURLPATH{"view" path="Cartoons.EvilMonkey" rev="1"}%= will expand to =%SCRIPTURLPATH{"view" path="Cartoons.EvilMonkey" rev="1" }%=
   * Example: =%<nop>SCRIPTURLPATH{"edit" path="Cartoons.EvilMonkey" t="%<nop>GMTIME{"$epoch"}%"}%= expands to =%SCRIPTURLPATH{"edit" path="Cartoons.EvilMonkey" t="%GMTIME{"$epoch"}%"}%=
   * Example: =%<nop>SCRIPTURLPATH%= expands to =%SCRIPTURLPATH%=
   * Example: =%<nop>SCRIPTURLPATH{script}%= expands to =%SCRIPTURLPATH{script}%=
   <div class="foswikiHelp">%X% The =edit= script should always be used in conjunction with a =t="%<nop>GMTIME{"$epoch"}%"= parameter to ensure pages about to be edited are not cached in the browser<p></p>
   %X% See =[[VarSCRIPTURL][SCRIPTURL]]= if you expect to need the protocol and host e.g. if you are saving the HTML of the page and using it on a different host.<p></p>
   %X% The 'old' way of building URLs using =SCRIPTURLPATH= involved concatenating the web and topic names to the =SCRIPTURLPATH= e.g. =%<nop>SCRIPTURLPATH{"script"}%/Cartoons/EvilMonkey=. This practice is *strongly* discouraged, as it does not correctly handle encoding of the parts of the URL. At the first opportunity you should replace such URLs with the equivalent %<nop>SCRIPTURLPATH%{"script" path="Cartoons.EvilMonkey"}%, which will handle URL encoding for you.</div>
